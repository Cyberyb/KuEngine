# v0.2 开发执行计划（RenderGraph Alpha）

## 1. 文档目的

本文件用于固化 v0.2 的阶段化开发路径、验收门禁与任务看板，避免范围漂移，确保从 v0.1.x 基线平滑进入 RenderGraph Alpha。

---

## 2. 阶段0冻结结果（2026-04-14）

### 2.1 v0.2 范围（In Scope）

- RenderGraph Alpha 最小数据模型：Pass 节点、资源句柄、读写声明。
- compile 阶段：依赖分析、拓扑排序、基础 barrier 计划生成。
- execute 阶段：按编译结果执行多 Pass（不再依赖注册顺序）。
- 至少 3 个 Pass 正确执行（含 swapchain 路径）。
- 可观测性最小闭环：编译日志、执行顺序可视化摘要。

### 2.2 v0.2 非目标（Out of Scope）

- glTF 场景、材质系统、复杂资源导入。
- Compute 管线与 GPU profiling 完整体系。
- Capture/Replay 与批量回归框架。
- 大规模性能优化（v0.2 以正确性为先）。

### 2.3 阶段0 DoD（Definition of Done）

- 范围冻结完成并文档化（本文件 + 总览文档同步）。
- v0.2 验收门禁清单已定义。
- 任务看板与里程碑已定义。
- 风险与回滚策略已定义。

---

## 3. v0.2 验收门禁清单

### 3.1 功能门禁

- 至少 3 个 Pass 在同一帧图中稳定执行。
- RenderGraph 执行顺序与依赖拓扑一致。
- 自动 barrier 在基础颜色输出路径可用。
- resize 后可稳定恢复（不崩溃、不黑屏）。

### 3.2 工程门禁

- Debug 构建通过。
- 现有 Triangle/Cube 示例可继续运行。
- 新增最小单元测试覆盖：拓扑排序与循环依赖检测。
- 核心文档与实现保持一致（发布前文档一致性检查）。

### 3.3 可观测性门禁

- compile 输出可读的 Pass 顺序摘要。
- execute 输出可读的资源状态/屏障摘要。
- 错误日志能区分声明期、编译期、执行期问题。

---

## 4. 阶段化开发步骤

### 阶段1：最小数据模型

目标：建立 RenderGraph 的核心结构，不破坏现有 Pass 生命周期。

交付：
- RenderGraph/GraphBuilder/ResourceHandle 基础类型。
- Pass 读写声明入口。
- 与现有 RenderPipeline 的兼容桥接层。

### 阶段2：依赖编译器

目标：把声明信息转换为确定执行计划。

交付：
- 资源访问图构建。
- 拓扑排序。
- 循环依赖检测与错误提示。
- 基础 barrier 计划结构。

### 阶段3：执行器对接 RHI

目标：按编译结果录制和执行命令。

交付：
- execute 按计划驱动。
- barrier 调用接入 CommandList。
- swapchain 相关布局转换链路稳定。

### 阶段4：三 Pass Alpha 场景

目标：完成 v0.2 关键验收样例。

交付：
- 至少 3 Pass 的演示路径。
- UI 参数仍可调，且与执行路径一致。
- resize/recreate 场景验证通过。

### 阶段5：可观测与测试补强

目标：形成可调试、可回归的最小闭环。

交付：
- 编译期/执行期日志补全。
- UI 调试摘要面板（最小版）。
- 最小自动化测试覆盖与回归脚本说明。

### 阶段6：发布收尾

目标：完成 v0.2 Alpha 发布前清理。

交付：
- 文档同步（总览、RenderPass、使用说明、工作日志）。
- 门禁逐项勾选通过。
- CHANGELOG 更新。

## 6. 风险与应对

- 风险：接口大改导致示例失效。
  - 应对：保留兼容桥接层，逐步迁移。
- 风险：barrier 规则复杂导致开发停滞。
  - 应对：v0.2 先支持基础颜色路径，后续迭代扩展。
- 风险：文档与代码再次漂移。
  - 应对：每个阶段结束执行文档对齐检查。

## 8. 阶段1完成记录（2026-04-14）

### 8.1 已交付

- 新增 RenderGraph 最小模型（Pass 节点、资源句柄、读写声明）。
- RenderPipeline `compile` 已切换到 `setup(RenderGraphBuilder&)` 桥接路径。
- 保留旧 `setup()` 兼容能力，未迁移 Pass 可继续运行。

### 8.2 当前限制

- `RenderGraph::compile()` 仍为注册顺序执行。
- 尚未启用拓扑排序、循环依赖检测和 barrier 计划生成（阶段2目标）。

## 9. 阶段2完成记录（2026-04-14）

### 9.1 已交付

- RenderGraph 编译器支持资源冲突依赖边构建。
- 支持显式依赖声明（`dependsOn(passName)`）。
- 拓扑排序生成执行序，替代阶段1顺序执行。
- 循环依赖检测与错误提示已接入。
- 基础 barrier 计划结构已生成（RAW/WAR/WAW）。
- 新增单元测试覆盖：依赖排序、缺失依赖、循环依赖、资源依赖与 barrier 计划。

### 9.2 当前限制

- barrier 计划仅为编译期数据，尚未驱动实际 RHI barrier 插入（阶段3目标）。
- CMake Tools 在当前会话无法完成配置，尚未通过工具链完成自动测试执行。

## 10. 阶段3完成记录（2026-04-14）

### 10.1 已交付

- RenderPipeline 执行阶段已接入 barrier 计划消费逻辑。
- 新增外部图像绑定能力（按资源名绑定 `VkImage` 与当前布局）。
- barrier 发射已接入 `CommandList::imageBarrier`。
- `CommandList::imageBarrier` 增强了常见布局访问掩码填充。
- Triangle/Cube 示例已统一使用 `CommandList::imageBarrier` 处理 swapchain 布局转换。

### 10.2 验证结果

- 使用 CMake Preset 完成全量 Debug 构建（库 + 示例 + 测试）通过。
- `ctest --preset debug` 通过。

### 10.3 当前限制

- 当前示例仍采用“单次 BeginRendering 包裹全部 Pass”的执行模型。
- 多 Pass 间更细粒度 barrier 与附件切换将在阶段4样例中继续压测与收敛。

## 11. 阶段4完成记录（2026-04-14）

### 11.3 当前限制

## 12. 阶段5完成记录（2026-04-14）

### 12.1 已交付

- `RenderPipeline::drawUI` 新增 `RenderGraph Debug` 面板，包含：
  - 编译摘要（pass/resource/dependency/barrier 数量）
  - 依赖边列表与 barrier 计划列表
  - 最近一帧执行摘要（planned/applied/skipped）
- `RenderPipeline::execute` 新增 barrier 执行事件记录（应用或跳过原因）。
- RenderGraph 单测补强：
  - 新增 hazard 类型覆盖（RAW/WAR/WAW）
  - 新增 external 资源标记行为验证
- 新增最小回归说明文档：`docs/usage/v0.2-regression-checks.md`。

### 12.2 验证结果

- Debug 构建通过。
- `ctest --preset debug` 通过。

### 12.3 当前限制

- 调试面板目前为全局展示，尚未提供筛选和导出能力。
- 执行期摘要目前聚焦 image barrier，buffer 资源链路放在后续版本扩展。

### 12.4 收尾补充（2026-04-15）

- 已补充动态渲染区间保护：当 `RenderPipeline::execute` 在 rendering scope 内运行时，遇到真实布局转换需求会跳过 barrier 并记录原因，避免触发 Vulkan VUID-08719。
- 新增执行上下文开关：`setExecuteInsideRendering(bool)`，示例已统一声明为 `true`。
