# v0.2 最小回归检查清单

本文档提供 v0.2 阶段5的最小回归流程，目标是快速确认 RenderGraph 编译与执行链路没有回归。

## 1. 环境前提

- 已安装 Visual Studio 2022（含 C++ 工具链）
- 已安装 Vulkan SDK（含 glslc）
- 仓库根目录包含 `CMakePresets.json`

## 2. 构建与测试

在仓库根目录执行：

```powershell
cmake --preset vs2022-vcpkg
cmake --build --preset debug
ctest --preset debug
```

期望结果：

- 配置成功
- Debug 构建成功
- `test_core` 通过

## 3. 示例回归（Alpha3Pass）

```powershell
cmake --build --preset debug --target Alpha3PassApp
```

启动 `build/cmake-tools/vs2022-vcpkg/bin/Debug/Alpha3PassApp.exe` 后检查：

- 可看到 `RenderGraph Debug` 窗口
- Compile 摘要包含 pass/resource/deps/barriers 计数
- Barrier Plan 和 Dependencies 列表可见
- Last Execute 在每帧更新，且 `skipped-noop` 或 `skipped-in-rendering` 计数可见（用于避免动态渲染区间非法 image barrier）

## 4. 常见回归信号

- Compile 阶段依赖边数量突然为 0（检查 Pass 是否声明 `read/write/dependsOn`）
- Last Execute 中 `skipped-unbound` 持续上升（检查是否调用 `bindExternalImage`）
- Last Execute 中 `skipped-in-rendering` 持续上升且出现画面异常（说明存在需移到 BeginRendering 之前的真实布局转换）
- `ctest` 中 RenderGraph 相关测试失败（优先检查 `RenderGraph::compile`）
